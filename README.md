# 八角星線上集點頁面與流程規劃

## 檔案

- `index.html`：可直接開啟的互動原型，包含報名集點卡、每位參加者專屬 QR、關主核發菱形、後台表格、抽獎名單、流程規劃。
- `embed-snippet.html`：可貼到網站或課程頁面的 iframe 範例。

目前 `index.html` 使用瀏覽器本機 `localStorage` 模擬資料，適合確認流程與介面。正式活動需要接 Google Sheet、Airtable、Supabase、Firebase 或學校網站後台，讓所有手機看到同一份資料。

公開網址：https://laierin615.github.io/amis-star-checkpoint/

主控台密碼：`8888`

現場簡化版：

- 關主口令：第幾關就輸入幾，例如第二關輸入 `2`、第八關輸入 `8`。
- 參加者代碼：三碼數字，例如 `001`、`002`、`003`；手動輸入時只打 `1` 也會自動對到 `001`。
- 舊版參加者代碼 `NP115001` 仍可掃描或輸入，系統會自動轉成 `001`。
- 舊版關主口令 `2102` 到 `2108` 仍可使用，方便已印好的資料延續。

## 推薦流程

### 1. 活動前

1. 建立八個關卡資料：關卡名稱、關主姓名、關主手機、核發菱形口令、是否啟用。
2. 活動開始前或現場入口先收集姓名與身分別，身分別固定為「學生、來賓、家長、師長」。
3. 完成報名後，系統自動產生該參加者獨一無二的 QR code；QR 內容是帶有參加代碼的報名集點卡網址。
4. 產生八位關主的核發網址：每位關主只看得到自己的關卡，口令直接使用關卡號碼。
5. 印出或發送參加者 QR 卡，關主手機先測試掃碼與核發菱形。

### 2. 闖關中

1. 第一關為 Google 表單：參加者開啟 `https://forms.gle/1ZsEmkVoPNFVHNSFA` 後，系統自動核發第一關菱形。
2. 其他關卡完成任務後，出示自己的專屬 QR 或參加者代碼。
3. 關主用自己的手機開啟「關主給點」頁。
4. 關主掃參加者 QR 後，系統自動帶入參加者代碼；也可手動輸入代碼。
5. 關主確認任務通過後，按「核發菱形」。
6. 系統檢查：關主口令是否正確、參加者是否存在、該關是否已經給過點。
7. 成功後該菱形亮起，八個菱形集滿後組成阿美族八角星圖騰。

### 3. 收尾

1. 主控台查看完成八個菱形、缺少關卡、重複或異常紀錄。
2. 匯出完整後台表格，或只匯出完成八角星的抽獎名單。
3. 關閉關主給點權限，避免活動後繼續被修改。

## 關主如何給點

推薦採用「關主掃參加者 QR」：

1. 關主選擇自己的關卡，口令輸入該關卡號碼。
2. 參加者完成任務後，關主掃參加者專屬 QR。
3. 畫面顯示參加者姓名、身分別、目前完成關卡。
4. 關主按「核發本關菱形」。
5. 系統寫入 `participant_id + station_id + host_id + timestamp`。

這個模式比「參加者掃關卡 QR」可靠，因為參加者不能自己觸發核發菱形。若現場希望參加者掃關卡 QR，建議加一道關主當場告知的一次性口令，或每 10 到 15 分鐘更換一次關卡口令。

## 資料表設計

### participants

| 欄位 | 說明 |
| --- | --- |
| `participant_id` | 參加者唯一代碼，例如 `001` |
| `name` | 姓名 |
| `group_name` | 身分別：學生、來賓、家長、師長 |
| `qr_url` | 參加者專屬 QR 對應網址 |
| `created_at` | 建立時間 |
| `raffle_eligible` | 是否完成八個菱形，可由系統依紀錄自動計算 |

### stations

| 欄位 | 說明 |
| --- | --- |
| `station_id` | 關卡代碼，例如 `s1` 到 `s8` |
| `station_name` | 關卡名稱 |
| `host_name` | 關主姓名 |
| `host_pin` | 關主口令，正式版應雜湊或放在後端 |
| `enabled` | 是否開放給點 |

### stamps

| 欄位 | 說明 |
| --- | --- |
| `stamp_id` | 單筆核點紀錄 |
| `participant_id` | 參加者代碼 |
| `station_id` | 關卡代碼 |
| `host_id` | 關主代碼 |
| `created_at` | 核點時間 |
| `device_note` | 裝置或備註，方便查核 |

`stamps` 需要設定唯一限制：同一個 `participant_id + station_id` 只能存在一筆，這樣可以阻擋同一關重複核發菱形。抽獎名單由 `stamps` 中完成 8 筆不同關卡紀錄的參加者自動產生。

## 快速上線方案

### 方案 A：Google Sheet + Apps Script

適合學校活動快速使用。資料存在 Google Sheet，Apps Script 提供讀寫 API。優點是容易維護、可直接匯出表格；缺點是多人同時大量操作時需要測試效能與權限。

### 方案 B：Supabase / Firebase

適合活動人數多、需要即時同步與登入權限。優點是資料同步穩、可設定權限；缺點是需要較完整的後端設定。

### 方案 C：純靜態展示

只適合展示或試流程。每台手機資料不會互通，不能作為正式活動核點系統。

## 現場備援

- 每位關主準備一張紙本補登表：參加者代碼、時間、是否通過。
- 主控端準備一支可分享網路的手機或行動網路。
- 若網路中斷，關主先紙本記錄，活動後由主控端批次補登。
- 參加者 QR 卡建議同時印出代碼，掃碼失敗時可手動輸入。
